# 냉장고 레시피 추천 프로젝트 (가제: 냉장고를 부탁해 🍳)

MVP 개발 과정을 기록하고, 아이디어를 현실로 만드는 프로젝트입니다.

## 1. 프로젝트 개요 

"오늘 뭐 먹지?"라는 고민과 냉장고 속 방치되는 식재료 문제를 해결하기 위한 서비스입니다.

사용자가 현재 냉장고에 보유한 재료를 입력하면, **1) 현재 바로 만들 수 있는 요리**와 **2) 1~2가지의 재료만 추가로 구매하면 만들 수 있는 요리**를 추천합니다. 레시피는 검증된 자취 요리 유튜버의 영상으로 연결하고, 부족한 재료는 쿠팡 등 커머스로 연결하여 수익 모델까지 검증하는 것을 목표로 합니다.

## 2. 기술 스택 (Tech Stack)

본 프로젝트는 3개월 내 **빠른 MVP 완성**과 **비즈니스 로직 검증**에 초점을 맞추어 다음과 같은 기술 스택을 전략적으로 선택했습니다.

- **프론트엔드/백엔드**: Bubble.io 

 - **선택 이유**: 개발 경험이 적은 상태에서 복잡한 코딩 학습 시간을 줄이고, 아이디어의 핵심 로직(DB 설계, 워크플로우) 구현에 집중하기 위함.

- **데이터베이스**: Bubble 내장 DB

- **데이터 수집**: Python, ChatGPT API (유튜브 레시피 데이터 반자동화)

- **프로젝트 관리**: GitHub, Google Sheets

## 3. 구현 기능 

- **재료 관리**: 사용자가 보유한 재료를 DB에 등록 및 삭제

- **즉시 추천**: 보유 재료만으로 만들 수 있는 레시피 목록 필터링

- **구매 유도 추천**: 1~2개의 재료가 부족한 레시피 목록 필터링

- **레시피 연결**: 추천된 레시피를 클릭 시, 해당 요리의 유튜브 영상으로 연결

- **커머스 연동**: 부족한 재료를 쿠팡 링크로 연결하여 수익성 검증

    1.  **재료의 양/단위 무시:** MVP 단계에서 '양(g, ml)'은 제외하고 **재료의 '존재 유무'** 로직에만 집중하기로 결정. (UX와 구현 난이도를 낮추기 위한 전략적 판단)
